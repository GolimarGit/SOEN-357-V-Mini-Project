Usability Testing

User testing sessions will allow for the prototype to reflect on real use cases. The usability test should focus on the placement of components, the ease of use, and feature availability within the application. It should identify weaknesses in the current UI and offer improvements to enhance the UX.

Participants representing the personas' background will perform given tasks covering the core functionalities of the application:

Registration and Profile Setup

  • Create an account and set basic preferences
  • Add a dependent to the profile
Profile Management
  • Update personal information
Appointment Booking and Management
  • Select a doctor
  • Book an appointment for themselves and another for their dependent
  • Reschedule or cancel an appointment
Medication Reminder Setup
  • Add a medication
  • Log a missed or taken dose
  • Delete a medication reminder

Data collection will involve participant feedback through think-aloud protocol and a post-test interview, where the participant will provide feedback on the usability and their satisfaction, and will involve tests where observers must analyze metrics such as:

  • Task completion rate
  • Execution time
  • Errors made
  • Heatmap and interaction tracking

Once the post-test interview has been recorded, the data will be categorized into usability issues and ranked by severity, which will then be mapped to the user journey. Enhancements will then be focused on improving the core functionalities of:

  • profile and dependent management
  • appointment booking process
  • medication reminder interactions
as well as including additional features suggested by the participants during the post-test analysis.
